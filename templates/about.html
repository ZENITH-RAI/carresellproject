About this project

AutoValue is the front end for your final-year used-car resale FYP: a price estimator paired with a custom ML pipeline (SGD regressor, feature engineering in Python).

Front-end stack

The site lives in frontend/: semantic HTML, one shared stylesheet (css/styles.css), and small scripts—js/main.js for navigation and form submit handling, js/vehicleAutocomplete.js for brand/model search. There is no React or build step; you can open the pages from any static host or run python -m http.server inside frontend/ during development.

Brand and model data (UsedCars.csv)

Listing suggestions are not hand-typed: scripts/build_vehicle_catalog.py reads UsedCars.csv and writes frontend/data/vehicle_catalog.json. Splitting matches app.py: the first token of name is brand, the remainder is model. Each brand lists the distinct models seen in the data, plus an Other option at the end for rare trims that map to the model bucket in training.

After you update the CSV, regenerate the JSON from the repository root:

python scripts/build_vehicle_catalog.py

Estimator field names

The form uses the same names as the server expects:

  • brand, model, year, km_driven
  • fuel, transmission, seller_type, owner
  • mileage, engine, max_power, seats

On the server, brand_model is derived as brand + "_" + model; the front end does not need a separate field for that.

Next step for integration

Replace the submit handler in js/main.js with a fetch POST to your API, then render the formatted NPR string in the result card. Until then, the UI validates input and shows a short loading state.
